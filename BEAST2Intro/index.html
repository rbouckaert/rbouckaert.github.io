BEAST2 Requirements

  • Plugin / package architecture
  • Checkpointing / restarting analyses
  • Self documenting code
  • Automatic software testing (junit, fest, Hudson)
  • Generalisation of existing models

BEAST Objects & Inputs

  • Every object implements BEASTInterface
  • BEASTObject archetypical implementation
  • Object connect with other objects through Inputs
  • Object graph is a DAG
  • BEAUti: editor of the DAG
  • BEAUti template: provide subgraph + rules on how to connect
